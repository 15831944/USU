Internet Programming Lecture 5100.41

  1. ActiveX Controls in Web Pages
    1. Just as applets can be displayed in web pages on Java-enabled browsers, so also can ActiveX controls be displayed.
    2. The Microsoft Internet Explorer does this automatically, while Netscape requires a plugin.
    3. The web page serves as a container for the control.
    4. You can also use the ActiveX Control Test Container from the Tools menu to display your control. Be sure to register the control first from the same menu, so you can insert it into the test container.
  2. Security controls
    1. Unlike Java, ActiveX controls are not restricted to a sandbox. Instead, they are given full access to all system resources.
    2. However, the browser will warn users who have specified at least medium levels of security that a control may not be safe.
    3. It is possible for control authors to eliminate this warning, at the cost of registering it as safe for scripting and initializing. Code is digitally signed, so that if something goes wrong, the user can discover whom to blame.
  3. Advantages of ActiveX controls over Java applets
    1. They only need to be downloaded once.
    2. They are usually quite a bit faster.
    3. Controls can fire events to the container.
    4. Controls can interact with Word, Visual Basic forms, and Visual C++ programs.
    5. Controls have greater flexibility of action, since they operate outside the sandbox.
    6. Controls can be licensed, providing an income flow to their authors.
  4. Advantages of Java applets over ActiveX controls
    1. Typically, ActiveX controls are not as portable as Java applets.
    2. ActiveX controls aren’t as safe.
    3. ActiveX controls are stored permanently on user disks and referenced permanently in user registries.
    4. ActiveX controls are typically larger than corresponding applets.

  6. Other Visual C++/Internet connections
    1. The CInternetConnection and CInternetFile classes allow us to establish http, ftp, or gopher sessions, and then send and receive information.
    2. The CSocket and CAsynchSocket classes allow us to work directly with sockets via the Winsock specification.
    3. It is possible to select MAPI support through the AppWizard, thus allowing documents to be sent automatically through e-mail using the Messaging API.
    4. The ISAPI classes (Internet Server API) extend the hypertext transfer protocol (http) and also permit the use of filtering. There is an ISAPI Extension Wizard to simplify ISAPI application generation.
