Text Components and Trees Lecture 5100.16

  1. JTextComponent
    1. This is the base, abstract class for all other text objects (figure 7.21)
      1. JTextField and JTextArea
      2. JPasswordField
      3. JEditorPane
      4. JTextPane
    2. While abstract, it does implement a great deal of the functionality of all other text objects.
      1. Standard operations like cut, copy, and paste.
      2. Editor commands like move to end or beginning of line or text
      3. Keymaps which associate specific keystrokes with editor commands
      4. Cursor and caret position
      5. Document model and document events - DefaultStyledDocument
      6. Text selection
      7. Loading from and saving to character streams
      8. Associating points in the view to text positions in the model
  2. JTextField
    1. Much like the AWT TextField
    2. Constructor
      1. Document
      2. String
      3. Column size
    3. Horizontal alignment
    4. Horizontal visibility
    5. Scroll offset
    6. Action command
  3. JPasswordField
    1. Pretty much like text field, but echoes all characters with an echo character, which can be set and gotten. If the echo character is not set, there is no echo.
    2. The getText method is replaced by getPassword.
  4. JTextArea
    1. Being multi-line, there is a possibility of line wrapping, if the method setLineWrap has been called. Lines will be wrapped at either word or character boundaries, as determined by setWrapStyleWord.
    2. Constructor
      1. Document
      2. String
      3. Row and column size
    3. Line operations
      1. getLineCount
      2. getLineEnd(Start)Offset to get character positions within the document
      3. getLineOfOffset to reverse, that is, go from position to line number
    4. insert and append and replaceRange
    5. setTabSize
  5. JEditorPane
    1. Able to work with text that is HTML or RTF encoded, as well as straight ASCII text.
    2. Constructor
      1. URL
      2. MIME type
      3. String
    3. Initialization through setPage, setText, or read
    4. Editor kits
    5. Works with hyperlinks.
  6. JTextPane
    1. Able to work with multiple text styles (e.g. fonts) within a single document.
    2. Can contain images or other objects.
    3. Can be constructed with a standard document model.
    4. Styles
      1. Style Context is the first thing you need. Call StyleContext.getDefaultStyleContext
      2. Then get a default style and give it a name with addStyle.
      3. Then edit it using setFontFamily, setFontSize, setItalic, etc.
      4. Then add it to the text pane with addStyle.
    5. Text is inserted with a style, using insertString, for instance.
    6. The attributes (style attribute values) of a character or paragraph can be found.
  7. JTree
    1. This is a huge class, which provides the view for tree structures like directory trees, etc.
    2. Auxiliary classes
      1. TreePath
      2. TreeNode
      3. TreeModel
      4. TreeCellRenderer
      5. TreeModelEvent
      6. TreeExpansionEvent
      7. TreeSelectionModel
      8. TreeSelectionEvent
      9. TreeWillExpandEvent
    3. Constructor.
      1. The root may be visible, or not.
      2. TreeNode object – displays as root
      3. Hashtable – root not displayed
      4. Object array – root not displayed
      5. Vector – root not displayed
      6. TreeModel – tree displayed
    4. Nodes are identified by row or by path. They can be expanded or collapsed. They can be tested to determine whether they are expanded.
    5. Points in the view can be translated to paths or rows, and vice versa.
    6. All sorts of selection operations are possible.
