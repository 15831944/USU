Swing Data Models Lecture 5100.24

  1. The BoundedRangeModel represents a single value or an extent of values within a range of integers. Minumum <= value <= value+extent <= maximum
    1. Events generated: change events
    2. Model for: slider, scrollbar, progressbar
    3. No constants.
    4. Methods to get and set the four parameters above, as well to add and remove change listeners. There is also a "value is adjusting" mechanism.
  2. The ButtonModel represents the various aspects of the state of a button.
    1. Events generated: action, change, and item events
    2. Model for: AbstractButton, JButton, JMenuItem, JMenu, ButtonGroup
    3. No constants.
    4. Methods to add and remove listeners, get and set Mnemonics, get and set action commands, determine whether the button is enabled, armed, pressed, rolled over, or selected.
  3. The ComboBoxModel and MutableComboBoxModel model the information of a combo box (selection field with a dropdown selection list) which may or may not allow the user to modify the list. This is an extension of the ListModel interface.
    1. Events generated: action, item, list data events
    2. Models for JComboBox. Some combo boxes can be edited, and some can’t.
    3. No constants.
    4. Methods for ComboBoxModel: get and set selected item
    5. Methods for Mutable..: add and remove element, add or remove element at index.
  4. The SingleSelectionModel abstracts selection semantics away from specific structures like trees and lists. For instance, it permits only single selection among a collection.
    1. Events generated: change events
    2. Model for: tabbed pane, menu, popup menu, menubar
    3. No constants.
    4. Methods to add and remove listeners, get, set, or clear the current selection, and test whether there is a current selection.
  5. The TableModel and TableColumnModel allow modeling of tables (two dimenstional arrays) in which each column has a title and contains a separate type of object.
    1. Events generated: TableModelEvents: these have type inserted, deleted, updated; and TableColumnModelEvents: these have beginning and ending column information for when insertion or deletion affected several columns.
    2. Model for JTable and JTableHeader
    3. No constants.
    4. Table model methods to add and remove listeners, get the number of rows and columns, get the class of a column, get or set the value of a specific cell
    5. Table column methods to add and remove listeners, add, move, or remove columns, get column size, identify a column by name, index, or position, find selected columns, get or set a (list) selection model.
  6. The TreeModel is a data model for standard trees (not DAGs, as the book indicates). Each node in the tree can have as many children as it chooses to add.
    1. Events are TreeModelEvents, which contain path and child information.
    2. Model for Jtree
    3. No constants.
    4. Methods to add and remove listeners, get the number of children of a node, the child at a given index, or the index of a child, as well as get the root and test whether a node is a leaf. There is also a valueForPathChanged method which is called when the user has altered the value stored at a node.
  7. The TreeSelectionModel represents the selection state of a tree.
    1. Events are tree selection and property change events.
    2. Models the selection state of a tree.
    3. Constants for selection mode: single, contiguous, or discontiguous.
    4. Methods to add and remove listeners, get, set, add, remove, and clear selection paths, determine rows (levels) in which selections occur, test whether a path is selected, get the number of selections or whether anything is selected at all.
  8. The DocumentModel tries to abstract the general nature of documents, including that they are sequences of characters with a tree structure (e.g., paragraphs within sections within chapters).
    1. Events generated: document and undoable edit events. These are very complex event classes. Document events include change, remove, and insert, but undoable events include a lot of information.
    2. Model for JTextField, JPasswordField, JTextArea, JEditorPane, JTextPane.
    3. No constants
    4. Methods to add and remove listeners, find length, work with positions, find the largest structural elements (root elements), get properties, get text, insert and remove strings.
