Swing Buttons, Combo Boxes, File Choosers, Labels, Lists Lecture 5100.14

  1. Additional Swing Components Beyond AWT
    1. Panes
      1. JEditorPane understands particular formats and provides formatting;
      2. JLayeredPane
      3. JOptionPane makes it easy to show warnings, ask for confirmations or input, etc.;
      4. JScrollPane allows scrolling over a large view while maintaining column and row headers;
      5. JSplitPane provides two panes, horizontal or vertical, whose shared space can be adjusted by the user;
      6. JTabbedPane permits the user to click a tab and select a single component to occupy all the space in the container.
    2. Controls
      1. JprogressBar displays degree of task completion visually and as a percentage;
      2. JradioButton is part of a group of toggle buttons, only one of which can be selected at a time;
      3. JSlider behaves like a scroll bar, but has tick marks and doesn’t have arrow buttons;
      4. JToggleButton shows set/reset state by color change (default) or icon/text change;
      5. JPasswordField is just a TextField with an echo character set;
      6. JToolBar can be floated, can have margins, and can be either horizontal or vertical.
    3. Selection components
      1. JcolorChooser permits color selection using a variety of specialized tabbed panes;
      2. JComboBox is the standard editable text field with drop down selection list;
      3. JList is a generic selection list.
    4. View components
      1. JTree gives a view of an associated tree model;
      2. JViewport displays a rectangle within the underlying view;
      3. JTable displays data as a table with row and column headers.
    5. JToopTip
    6. In this section of the book, the author uses a new class, Box, and two layouts, Grid and Box.
      1. Box is essentially a panel with a box layout.
      2. The box layout permits easy nesting of vertical/horizontal arrangements of components
      3. The grid layout permits easy display of components in rows and columns.
  2. Buttons
    1. AbstractButton
    2. Implementation Buttons (figure 7.1)
      1. Button - standard
      2. CheckBox, RadioButton - two-state devices with different appearance and semantics
      3. MenuItem - appears in a menu as a selection
      4. CheckBoxMenuItem, RadioButtonMenuItem- toggled menu selection
      5. Menu - drops a menu from a menubar
      6. ToggleButton - an abstraction
    3. General Button Behavior
      1. Buttons have several states: enabled, selected, and rollover enabled. Additionally, toggle buttons have a set/reset state.
      2. Buttons can display text and/or icons. The position of the text with respect to the icon, and the alignment of the text/icon pair in the available space, can be set or gotten.
      3. Icon is an interface. The ImageIcon class implements it, and allows easy construction of icons from gif files.
      4. Mnemonics are keys which, when the AbstractButton is visible, trigger the action of the button. A mnemonic should be a letter in the text of a button, which will be underlined. If the letter appears multiple times, the first occurrence is underlined.
      5. The setFocusPainted method allows us to indicate whether a button will be highlighted when it has the keyboard focus. If it has focus, then the enter key will also trigger the action.
  3. Combo Boxes
    1. These are selection components which allow us to display the single selected item. When the drop down list is displayed, we can select another item. The list will then disappear, and the new selected item will be displayed. If a combo box is editable, then we can also select by typing the name of an item on the list.
    2. Combo boxes can be constructed either from a model, or from an array of objects, or from a vector.
    3. In a combo box we can add items to the end, insert them at a particular position, set the number of items that can be seen without scrolling, and get or set the item selected.
  4. File Choosers
    1. These are the familiar file selection boxes which permit us to navigate directories, display files according to a filter, and do something with the selected file(s), usually open or save them. It is a very complex class, with a large number of methods.
    2. The action to be taken in selecting a file can be whatever you wish. This action is triggered by clicking the Approve button. Generally, we may have a single event handler for that event, so it may be necessary to determine what kind of file chooser is displayed. The may be done by means of the getDialogType, getApproveButtonText, or getDialogTitle methods.
    3. The event handler will need to know which file(s) were selected. The can be done by the getSelectedFile and getSelectedFiles methods. These return objects of type File, so the getName method may be necessary to obtain the actual name string of the file. You can also get the description of a file.
    4. The event handler will need to know the directory being displayed, using getCurrentDirectory.
    5. We can set a file selection mode, permitting the selection of files only, directories only, or both.
    6. Sometimes a File Chooser being displayed becomes outdated. It can be updated by the rescanCurrentDirectory method.
  5. Labels
    1. Labels provide explanatory text and/or icons in the GUI. They can't be selected or receive focus.
    2. Labels can be constructed with a text, an icon, and a horizontal alignment within the area which contains them.
    3. Labels can be aligned by calling setHorizontalAligment and setVerticalAligment with one of the alignment constants, leading, trailing, top, bottom, left, right, or center.
    4. They can also be aligned with setAlignmentX, a method which they inherit from Component. The argument is a float between 1.0 (left) and 0.0 (right).
  6. Lists
    1. These are selection devices for all sorts of objects, like strings, images, buttons, etc.
    2. Lists can enforce single selection, interval selection (some number of contiguous items), or multiple interval selection (multiple contiguous stretches, each within a set region). Standard multiple selection is achieved by letting each element be a different region.
    3. Lists can be constructed from an array, a vector, or a data model.
    4. There are calls for controlling the number of items visible without scrolling, whether a specific item is visible, what the selection highlight color is, etc.
    5. Methods exist to tell what item(s) have been selected, or to select other(s). Getting can be done by retrieving index positions or objects.
    6. There are also methods for obtaining pixel values of selections and changing these to indexes.
