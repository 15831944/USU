Property Pages, Sheets, and Wizards Lecture 5100.37

  1. Property Sheets and Pages
    1. Property sheets are dialogs which contain other dialogs representing pages.
    2. The user selects a particular property page by clicking a tab.
    3. Pages are dialogs, and should have "Child" and "Thin" chosen in the style section of the resource editor property sheet.
    4. Pages shouldn’t have a system menu. This is the menu at the top-left, which has things like restore, move, minimize, close, etc.
    5. Each page has a surrogate object derived from CPropertyPage, with members for each control that appears on the property page.
    6. The surrogate for the property sheet as a whole is a CPropertySheet, with members for each page type, that is the specialized type derived from CPropertyPage.
    7. In the constructor for the property sheet, call AddPage for each of these individual pages.
    8. Display the sheet just like a dialog by calling DoModal
  2. Wizards
    1. Wizards are just property sheets which are navigated by Back, Next, and Finish buttons. They require the user to follow some specific, linear course in setting the values in the property pages, instead of being able simply to do this at random.
    2. Property pages for a wizard are created as before for property sheets.
    3. SetWizardMode is called to indicate that this is a wizard.
    4. A call to SetWizardButtons is made in page’s OnSetActive function. This establishes which of the next, back, finish, and cancel buttons will appear, and whether any of them is inactive.
    5. Each property page in the wizard has OnWizardBack, OnWizardNext, and OnWizardFinish handlers.
  3. There is an example project at the ftp site called junk3.
