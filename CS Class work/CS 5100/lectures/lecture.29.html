Visual C++ Environment and Controls Lecture 5100.29

A. The Visual C++ Environment

1. Similarities with Symantec Café

a. Project Organization

b. Different views

i. Class

ii. Resources (components)

iii. Files

iv. Documentation

c. Visual layout editor for components

d. Possibility of empty projects (Win32 application or Console)

e. Code sections marked with special comments – don’t change them

2. Differences from Symantec Café

a. Generates exe files rather than class files

b. Compiled, not interpreted

c. Not portable

d. Three general types of applications

i. Multiple Document Interface

ii. Single Document Interface

iii. Dialog

e. Code is harder to read and understand

f. We have to create surrogate variables for component values

g. Data is transferred directly between components and surrogate variables by DDX routines which are triggered by calls to UpdateData.

h. The class wizard is used for many things that happen automatically with the Symantec property window. It is also used in place of the interaction wizard.

i. Member functions like InitInstance perform much of the function of constructors, and functions like OnButtonClick take the place of listener methods.

j. Property window is in a different place, undocked window with pushpin

3. Some documentation isn’t present unless you use Internet Explorer or the CDROM.
  1. An Example Project

1. Make sure you have a base directory for your projects
  1. Specify you want a new project using File…New…MFC App Wizard
  2. fill in its name – this will create a new subdirectory of the base directory

3. Select Dialog as the application type

4. Choose included features and set title of dialog

5. Ask for comments and shared DLL

6. View project construction data

7. Open the dialog visual editor from the resource tab

8. Add components, changing properties like names, labels, etc.

9. Use the class wizard to add surrogate variables for screen components (resources)

10. Establish interactions with the class wizard through the message maps tab, including the customization of generated code, e.g., what to do when buttons are pushed

11. Open the code in the code editor and customize it, adding things like imports.

12. Run it.
