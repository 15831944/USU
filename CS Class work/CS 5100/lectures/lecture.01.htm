D.  Jargon

1.  A graphical user interface is one that makes extensive use of visual elements.

a.  W‑i‑m‑p: windows, icons, menus, pointing device.  This is the kind of interface we will usually be working on.

b.  Direct manipulation occurs when visual elements represent real world objects, and the pointing device represents your hand.  The goal is to make the computer disappear, and present a window onto the world.  A control panel is an example of a direct manipulation GUI.  We will also do some work with direct manipulation.

c.  Virtual reality occurs when you are provided stereoscopic vision and hearing, tactile senses, multiple control sensors, and virtual self-awareness.  It is beyond the scope of this course.

2.  Cognitive science is a field on the borders of computer science, psychology, and learning theory.  Many of the insights of cognitive science have contributed to improved user interface design.

3.  A style guide is usually provided for window systems.  It specifies all sorts of details, like the placement of a menu bar, the relationship of colors, the size of bevels, the direction of apparent lighting, etc.

4.  Following the style guide produces a “look and feel” in windows systems.  The open software foundation (OSF) or Motif style is evident in X‑windows systems, Macintosh, MS windows, and the Borland products.

5.  Typically, windows systems are constructed using elements from toolkits, like scroll bars, push buttons, text windows, etc.  We will be working with the AWT, MFC, and Motif toolkits.

F.  A small example using a Java UIDS

1.  Files for this example will be useful in doing your homework.  They are available at ftp.cs.usu.edu/pub/jones/cs5100/example1

2.  Classroom examples will be done using Symantec’s Visual Café version 4.0.  This is available in the 4th floor pc labs.

3.  Choose file…new project…AWT application application, then save as… some name in a newly created directory.

4.  In the object tab, delete the quit dialog, open dialog, and everything on the menubar except help.  Open the code by double-clicking the main file in the file tab, and put in window closing code.

5.  In the property list, rename the frame, and give it a different title and size.  Generally, you must hit the Enter key for these to take effect.  Notice how the code is changed when you change property values.

6.  In the visual editor drag a label, change its name, text, font and alignment.

7.  Add a button.  Change its name and background color.  Give it a “blank” label.

8.  Modify the About Dialog as you wish.

9.  Right click the button to add an interaction (event handler).  Choose action performed, set the background color, Color.black, finish.

10.  Modify the interaction code so that, in fact, clicking will cycle through four different colors.

11.  Modify the interaction code so that the button moves.

12.  Click the project window, File…Save All, then Project…Execute.
