Menus, ProgressBars, Sliders, Split/Tab Panes, ToolTips Lecture 5100.15

  1. Menu-related components
    1. Menubars
      1. Top level containers like JFrame and JInternalFrame automatically have menubars. They can be added to other containers as well. Menubars can contain menus only. Menubars have the following methods:
      2. The add method adds a menu to the menubar. The title used in constructing the menu is the name that appears on the menubar.
      3. getMenuCount tells how many menus appear in a menubar. The methods getMenu and getSubElements allow us to switch between the index (position) of the menu in the menubar and the menu itself.
      4. The method setHelpMenu allows us to specify which menu appears at the far right side of the menubar. It may not be available in JDK1.2.
    2. Menus
      1. Menus can display menu items, which includes not only the generic menu item, but also checkbox menu items, radiobutton menu items, and other (walking or cascaded) menus.
      2. Tearoff menus can remain displayed when the parent menu is closed. One form of menu constructor has a boolean argument to indicate whether the menu is a tearoff.
      3. Popup menus are menus which don't depend on a menubar to trigger their display. Usually they are displayed as a result of a right mouse button click. They must be displayed within the area of the parent.
      4. The add method adds another menu item. The insert method adds at a particular position. Another form of insert creates an entry representing an Action item as its action listener, since the Action interface extends ActionListener.
      5. addSeparator or insertSeparator will add a horizontal line for the purpose of visually dividing the menu into different sections.
      6. getItemCount and getMenuComponents return the number of components and an array of all components of a menu.
      7. getItem returns a specific menu item.
      8. doClick causes a menu selection programmatically.
    3. The setMnemonic method for menu items allows us to select the mnemonic by which a user can "click" a menu item from the keyboard.
  2. Progress Indicators
    1. These are the familiar objects used to indicate progress during a relatively slow operation. Typically, a JProgressBar is displayed within a dialog, which updates the bar periodically, and disappears when the operation is complete.
    2. Progress bars have a data member which is a value. The bar is redrawn when the value is set.
    3. There is a getValue to return the value, or a getPercentComplete, which indicates where the value stands between its minimum and maximum as a double number. The getString method returns the percent as a string.
    4. A JProgressBar constructor may specify orientation (horizontal or vertical), minimum and maximum values, or a data model.
  3. Sliders
    1. Sliders are somewhat like scroll bars, but they come with built-in tick marks and labels, indicating the range of values the slider may have. The "thumb" or button is more of a pointer, rather than the rectangle of the scrollbar.
    2. Slider methods include:
      1. Constructors which can set the orientation, initial, minimum, and maximum values of the slider, or a data model.
      2. Functions to display or suppress labels and tick marks.
      3. Functions to set or determine the "snap-to" state of the slider.
      4. Methods to deal with the inversion of a slider; that is, whether the maximum is displayed at the right or left side, top or bottom end.
      5. Functions to set the spacing of major and minor tick marks.
  4. Split Panes
    1. These are panes which can hold two objects, while allowing the user to determine how much real estate is occupied by one at the expense of another. The objects can be arranged side by side or top to bottom. When constructed with a "continuous" flag, this resizing is displayed as the panes are resized. Otherwise it happens only after resizing is complete.
    2. The stippled sash between the panes is dragged to change the relative size of the two objects. It can also be minimized and restored. The minimum size of each component can be set as well.
    3. The position of the divider can be initialized or reset programmatically. There is a method to reset the panes to their preferred relative sizes.
  5. Tabbed Panes
    1. These allow the same real estate to be shared by many containers, only one of which is visible at a time. The visible container is selected by the user clicking on a tab bearing its name.
    2. Tabs can be placed on any of the four sides of the tabbed pane. They can appear in multiple rows or columns.
    3. The add method can add at the end or insert at an index, and can allow the name of the object to appear on the tab or specify another name. Icons and tool tips can also appear on tabs.
    4. Tabs can be enabled or disabled.
    5. There are methods to find out how many tabs exist, which is currently selected, what the title of a given tab is, which tab contains a given title or icon, etc.
  6. Tool Tips
    1. The JToolTip component is perhaps the simplest of all Swing components. All JComponents can have a tool tip, which is set by a call to setToolTipText.
    2. Existing tool tips can have their text changed by a call to setTipText.
