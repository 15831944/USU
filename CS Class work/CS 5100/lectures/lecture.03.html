GUI Principles Relating To Users Lecture 5100.03
  1. Psychological issues
    1. Human mental processes
      1. Short-term memory permits instant, effortless access to about 7 chunks of information.
      2. Long-term memory permits associative, slow access to an unlimited amount of information.
      3. Problem solving uses awareness of a limited number of strategies.
      4. Decision making involves assessment of risk, effort, cost, benefit, etc. With the intention of maximizing some payoff formula while minimizing some payout formula.
      5. Attention and set relates to the volatility of short-term memory.
      6. Search and scanning uses perceptual and pattern matching skills, along with search strategies.
      7. Time perception tells us of the passage of time.
    2. Factors which may affect mental performance
      1. Arousal, fatigue, monotony
      2. Perceptual load
      3. Comfort, distraction
      4. Anxiety, fear
    3. Knowledge categories
      1. Syntactic knowledge relates to conventions and rules for which there is no good reason.
      2. Semantic knowledge relates to facts and procedural concepts. It is usually organized hierarchically, and may relate either to the problem domain or to computer systems.
    4. Familiarity reduces anxiety and fear while improving comfort. For this reason, the use of familiar metaphors has a positive effect on users. However, since users may read more into the metaphor than we have programmed, they can also be somewhat dangerous.
  2. How to create interfaces for users
    1. Start with measurable goals, such as
      1. How much time should be required for typical users to learn the system?
      2. How long should it take typical users to carry out a set of benchmark tasks?
      3. How many of what kinds of user errors can be permitted?
      4. How well do users retain their knowledge of system use after a day, or week, or month?
      5. How do users rate the system subjectively?
    2. Spend a significant time developing user profiles:
      1. Skill levels
      2. Training levels,
      3. Frequency of system use
      4. Tasks
      5. Preferences
      6. Priorities
      7. Willingness and tolerance
    3. Analyze tasks so as to determine atomic tasks, then put them together in scenarios or threads to implement all tasks.
    4. Build into your interface the following attributes:
      1. Consistency
      2. Power-user shortcuts
      3. Informative feedback; make sure that there are no hidden traps in the system state
      4. Dialogs which impose modality
      5. Disabling of choices which will generate errors, or at least warnings.
      6. Easy reversal of actions
      7. Reduction of short-term memory load
    5. In connection with error-blocking, consider how a wimp interface eliminates many of the common errors of command language interfaces:
      1. Modal errors (doing something inappropriate at the moment) are avoided by disabling menu selections and imposing modal dialogs.
      2. Description errors (forgetting how to formulate a command) are avoided by the menu mechanism.
      3. Activation errors (forgetting what to do next while in the midst of a sequence of actions) are avoided by cascading menus and modal dialogs.
      4. Capture errors (executing a memorized sequence of actions inappropriate for the moment) are avoided by disabling menu selections and modal dialogs.
    6. Carry out extensive prototyping
      1. Storyboarding
      2. Concept (paper)
      3. Design (paint)
      4. Animation (toolbook)
      5. Prototype (target UIDS)
      6. Final product
