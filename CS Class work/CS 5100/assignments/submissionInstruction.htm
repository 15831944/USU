First, I want students to delete all the intermediate files or executable files (ie. .class file for Java, .obj, .exe,... for MFC). However, I do want them to keep all the folders and the project files if they have any. I also want them to package up all the files into one zip file and name it, for instance hw3-JohnDoe.zip. That way I won't end up receiving a lot of files. Also, by keeping folders and project files (especially for MFC), it will be easier for me to rebuild.

 

  Second, for GUI assignments, students can design different action sequences in order to accomplish some tasks. If that's the case, I hope they can give a README file to explain it.  Then I won't spend a long time trying to figure out if the student's program really works.
